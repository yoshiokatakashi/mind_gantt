「次のアクション：プロトタイプ作成」とは、
**MVPの核となる “マインドマップ編集画面” を最小限動く形で作ること** を指します。

特に個人開発では「まず動くデモを作ってから詳細化する」ほうが圧倒的に速く進むため、この工程はとても重要です。

以下に **具体的に何をすればよいか** をまとめました。

---

# ✅ プロトタイプ作成でやること（具体的な作業内容）

## **1. Next.js プロジェクトを起動できる状態にする**

すでにリポジトリがあるので、以下が済んでいればOK：

* `npm install`
* `npm run dev` でローカルが立ち上がる

---

# **2. react-flow を使ったマインドマップの初期画面を作る**

これが最重要。

### ✔ react-flow をインストール

```
npm install reactflow
```

### ✔ ページを 1 枚作る（例：/editor）

* ノードが 1〜3 個だけ置いてある
* ドラッグで位置を動かせる
* 新しいノードを追加できる
* ノードを削除できる
  ※まだ DB 保存は不要

### 例：最低限のプロトタイプ構成

* `/editor/page.tsx`
* `components/MapEditor.tsx`
* react-flow でノードを並べるだけの画面

---

# **3. タスク追加・削除の最低限 UI を付ける**

ボタンを置くだけでOK。

例：

* 「ノード追加」ボタン → 画面中央に新規ノード
* ノードをクリックすると削除ボタンが出る

目的は操作感の検証。

---

# **4. デザインは Tailwind でざっくり**

プロトタイプなので細かい装飾は不要。

* 左上に簡易メニュー
* 右上にノード追加ボタン
* react-flow の画面が広めに表示されていれば十分

---

# **5. プロジェクト内部で “変換はまだしない”**

プロトタイプ段階では以下は **やらないほうが早い**：

* ガントチャート
* Supabase 保存
* 依存関係の生成
* プロジェクト構造の整理

今やるべきはあくまで **“マインドマップが直感的に作れるか” の検証**。

---

# 🎯 プロトタイプのゴール（できていればOK）

* [ ] react-flow のキャンバスが動く
* [ ] ノードが追加できる
* [ ] ノードが削除できる
* [ ] ノードがドラッグで動く
* [ ] 2〜3 ノードで簡易マインドマップが作れる
* [ ] ベース操作の手触りを確認できる

これだけでプロトタイプとして十分。
ここが完成してから、次に Supabase 保存やガント変換を組み込んでいきます。
